GOAL
Implement end-to-end apply behavior for MSG_PREF_APPL_LIST_SELECTED in Library/Pref/PrefAppl
so selecting an app creates a link in SP_APPLICATION. New links are created only in
SP_APPLICATION (never SP_TOP\\DESKTOP).

Deselecting an app removes all existing links targeting this app from SP_APPLICATION (and subfolders)
and SP_TOP\\DESKTOP (and subfolders). The existing link list should contain all currently existing links
in these folders.

Keep behavior robust on partial failures (continue processing remaining items).

STEP: Add reusable link operation helpers in scan/helper layer

Add a helper to create a link for an app record in SP_APPLICATION (exactly there, not in a subfolder)
using FileCreateLink. If a link with this name already exists, skip silently. No filename normalization
rules should be needed. You can create links with (pseudocode):
FileSetStandardPath(SP_APPLICATION);
FileCreateLink(record.PAAR_name,
		SP_SYS_APPLICATION,
		"DISTAPPL \ record.PAAR_name",
		1);

Add a helper to delete all scanned links that match an app token by iterating PrefApplLinkRecord
entries and removing each source link file (FileSetCurrentPath + FileDelete).
Use existing token comparison semantics (PrefApplTokensMatch) and keep MemLock/MemUnlock discipline.

STEP: Implement SELECTED logic itself

In Library/Pref/PrefAppl/prefappl.goc, implement MSG_PREF_APPL_LIST_SELECTED.

At the beginning of MSG_PREF_APPL_LIST_SELECTED, disable (...NOT_ENABLED...) the AppList, so that
no user interaction is possible while we are busy.

Read current list selection using the "selection" param. No call to PrefApplRescanApplicationLinks()
is needed before making changes, we should be current. We are using GIGBT_EXCLUSIVE_NONE with ApplList.
Add lastSelectedId as an instance field on PrefApplDialogClass (not global), initialized to GIGS_NONE.
Reset lastSelectedId = GIGS_NONE whenever the list is rebuilt from disk (MSG_PREF_APPL_REFRESH_LIST_FROM_DISK)
and on dialog initiate (MSG_GEN_INTERACTION_INITIATE), before status messages start flowing.
In MSG_PREF_APPL_LIST_SELECTED:
    If selection != GIGS_NONE: set lastSelectedId = selection and process as “selected”.
    If selection == GIGS_NONE: only process deselection if lastSelectedId != GIGS_NONE and < PrefApplGetApplicationCount(). Otherwise no-op.
    After handling that deselection path, immediately clear it: lastSelectedId = GIGS_NONE (prevents repeated deselect events acting on stale ID).
On any “record lookup failed / id out of range” path, clear lastSelectedId and return safely.

If the list item (application) just got selected:
    Check the scanned link list for a token match. If none exists, create exactly one link via helper (see above).
    make sure that AppList re-reads its monikers

If the list item (application) just got deselected:
    delete all links that are stored in the scanned links list matching that app token.
    remove their entries from the link list
    make sure that AppList re-reads its monikers

Make sure to run MSG_PREF_APPL_SHOW_ERROR for each operation.

Re-enable the AppList at the end, so that user interaction is possible again.

GENERAL CONSIDERATIONS:

C89 variable declarations at function top.
Small stack footprint (use handles/chunk arrays for variable-size buffers).
No recursion through link entries; preserve existing traversal safeguards.
Continue-on-error where possible; surface concise status text only when useful.
Keep create/delete logic in dedicated helpers in Library/Pref/PrefAppl/prefapplScan.goc so prefappl.goc stays orchestration-only.
Add comments to help future maintainers.
